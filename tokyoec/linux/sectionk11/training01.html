11.Infrastructure as Code

Ansibleの構築テキスト

事前作業

CentOS7_PROXY_DNSの仮想マシンで実行すること

  1. Nginxアンインストール
    yum remove -y nginx

CentOS7_WORDPRESSの仮想マシンで実行すること

  1. httpd,php,mariadbアンインストール
    yum remove -y httpd php mariadb
  2. ドキュメントルート内データの削除
    rm -rf /var/www/html/*
  3. mariadb関連ファイルの削除
    rm -rf /var/lib/mysql/*

2.Ansibleのインストール

CentOS7_PROXY_DNSの仮想マシンで実行すること

  1. ansibleのインストール
    yum install -y ansible
  2. バージョン確認
    ansible --version
  3. ansibleコマンドでansible-hostへpingを実行
    ansible 192.168.56.29 -m ping
  4. ansibleのhostsファイルにansible-hostのグローバルIPを登録
    vim /etc/ansible/hosts 
    ---------------------------------------------------------
#最終行にIPを追記
192.168.56.29
---------------------------------------------------------
    Esc + :wq
  5. ansibleコマンドでansible-hostへpingを実行
    ansible 192.168.56.29 -m ping
  6. 「/root/hostlist」ファイルを作成し、ansible-hostのグローバルIPを登録
    vim /root/hostlist 
    ---------------------------------------------------------
  #IPを記述
  192.168.56.29
---------------------------------------------------------
    Esc + :wq
  7. ansibleコマンドでansible-hostへpingを実行
    ansible 192.168.56.29 -m ping -k
  8. 鍵の作成
    ssh-keygen
  9. ansible-hostに鍵をコピー
    ssh-copy-id 192.168.56.29
  10. ansibleコマンドでansible-hostへpingを実行
    ansible 192.168.56.29 -m ping

3.Playbookの作成・WordPress環境構築

CentOS7_PROXY_DNSの仮想マシンで実行すること

  1. wordpress.ymlの作成
    vim wordpress.yml 
    ---
 - name: wordpress環境構築
   hosts: 192.168.56.29
   remote_user: root
   become: yes
   vars:
     wordpress_url: https://ja.wordpress.org/latest-ja.tar.gz
            
   tasks:
   - name: Apache（httpd）のインストール
     yum: name=httpd state=latest
            
   - name: サービスの起動
     service: name=httpd state=started enabled=yes

   - name: リポジトリ追加
     yum: name=epel-release state=latest

   - name: rpm配置
     get_url:
       url=http://rpms.famillecollet.com/enterprise/remi-release-7.rpm
       dest=/tmp/remi-release-7.rpm

   - name: rpmのインストール
     yum:
       name: /tmp/remi-release-7.rpm
       state: present

   - name: PHPのインストール
     yum: name="{{item}}" enablerepo=remi-php71 state=latest
     with_items:
      - php-mysql
      - php
      - php-devel
      - php-mbstring
      - php-pdo
      - php-gd
      - php-mcrypt
      - libmcrypt
            
   - name: タイムゾーンの設定
     ini_file: >
      dest=/etc/php.ini
      section=Date
      option=date.timezone
      value='"Asia/Tokyo"'

   - name: httpサービスの再起動
     service: name=httpd state=restarted enabled=yes
            
   - name: MariaDBインストール
     yum: name="{{item}}" state=latest
     with_items:
      - mariadb
      - mariadb-server
      - MySQL-python
      
   - name: サービスの起動
     service: name=mariadb state=started enabled=yes
            
   - name: wordpressのダウンロード
     get_url:
       url="{{ wordpress_url }}"
       dest=/tmp/wordpress.tar.gz
            
   - name: wordpressの展開
     unarchive: src=/tmp/wordpress.tar.gz dest=/var/www/html/ copy=no
            
   - name: wordpressの所有権をapacheに変更
     file: path=/var/www/html/wordpress/ owner=apache group=apache recurse=yes
            
   - name: httpサービスの起動
     service: name=httpd state=restarted
    Esc + :wq
  2. ansibleの実行
    ansible-playbook wordpress.yml
  3. http://192.168.56.29/wordpress/にアクセス

CentOS7_WORDPRESSの仮想マシンで実行すること

  1. mariadbのrootユーザのパスワード設定
    mysql -u root
    MariaDB [(none)]> update mysql.user set password=password('tokyoec') where user = 'root';
    MariaDB [(none)]> FLUSH PRIVILEGES;
    MariaDB [(none)]> exit
  2. WordPress用のDB作成
    mysql -u root -p
    Enter password:tokyoec
    MariaDB [(none)]> CREATE DATABASE wordpress;
    MariaDB [(none)]> GRANT ALL PRIVILEGES ON wordpress.*TO 'wordpress'@'localhost' IDENTIFIED BY 'wppass';
    MariaDB [(none)]> FLUSH PRIVILEGES;
    MariaDB [(none)]> exit
  3. wp-config.phpファイルの作成
    cd /var/www/html/wordpress
    vim wp-config.php
    右クリックでコピーした内容をペーストします。
    Esc + :wq
