6.ネットワーク管理

演習1 ネットワーク管理

1.net-toolsパッケージの確認

  1. カレントディレクトリをホームディレクトリに変更してください。

    cd
    pwd

  2. rootユーザに変更してください。

    su -

  3. net-toolsパッケージがインストールされているかどうかを確認してください。grepコマンドも利用してみましょう。

    rpm -qa | grep net-tools

  4. インストールされていない場合は、yumコマンドでインストールしてください。

    yum install -y net-tools

  5. インストールされたことを確認してください。

    rpm -qa | grep net-tools

  6. studentユーザに戻ってください。

    exit

2.ifconfigコマンド

  1. カレントディレクトリをホームディレクトリに変更してください。

    cd
    pwd

  2. rootユーザに変更してください。

    su -

  3. ifconfigコマンドを実行してインターフェイス情報を確認してください。

    ifconfig

  4. enp0s3に絞って、設定されているIPアドレス（IPv4,IPv6）、サブネットマスク、ブロードキャストアドレスを確認してください。

    ifconfig enp0s3

  5. ifconfigコマンドを使って、enp0s3を無効にしてください。

    ifconfig enp0s3 down

  6. 無効になっていることを確認してください。

    ifconfig enp0s3 ※IPアドレスが表示されないことを確認します。

  7. ifconfigコマンドを使って、enp0s3を有効にしてください。

    ifconfig enp0s3 up

  8. 有効になっていることを確認してください。

    ifconfig enp0s3 ※IPアドレスが表示されることを確認します。

  9. ifdownコマンドを使って、enp0s3を無効にしてください。

    ifdown enp0s3

  10. 無効になっていることを確認してください。

    ifconfig enp0s3 ※IPアドレスが表示されないことを確認します。

  11. ifupコマンドを使って、enp0s3を有効にしてください。

    ifup enp0s3

  12. 有効になっていることを確認してください。

    ifconfig enp0s3 ※IPアドレスが表示されることを確認します。

  13. studentユーザに戻ってください。

    exit

3.ipコマンドとrouteコマンド

  1. カレントディレクトリをホームディレクトリに変更してください。

    cd
    pwd

  2. rootユーザに変更してください。

    su -

  3. ipコマンドを使って、インターフェイス情報を確認してください。

    ip a
    ip addr
    ip addr show
    ip addr list
    ip address

  4. enp0s3に絞って、設定されているIPアドレス（IPv4,IPv6）、サブネットマスク、ブロードキャストアドレスを確認してください。

    ip a show/list enp0s3
    ip addr show/list enp0s3
    ip address show/list enp0s3

  5. ipコマンドを使って、ルーティングテーブルを確認してください。

    ip r
    ip route
    ip route show/list

  6. routeコマンドを使って、ルーティングテーブルを確認してください。

    route

  7. studentユーザに戻ってください。

    exit

4.hostnameコマンドとhostnamectlコマンド

  1. カレントディレクトリをホームディレクトリに変更してください。

    cd
    pwd

  2. rootユーザに変更してください。

    su -

  3. hostnameコマンドを使って、ホスト名を確認してください

    hostname

  4. hostnameコマンドを使って、一時的にホスト名を「students.com」に変更してください。変更できたことを確認してください。

    hostname students.com

  5. 仮想マシンを再起動して、再起動後にホスト名が元に戻っていることを確認してください。

    reboot
    hostname ※再起動後に実行する

  6. rootユーザに変更してください。

    su -

  7. hostnamectlコマンドを使って、恒久的にホスト名を「students.com」となるようにしてください。

    hostnamectl set-hostname students.com

  8. hostnamectlコマンドを使って、「students.com」に変更できたことを確認してください。

    hostnamectl status
    ※Static hostnameの箇所が「students.com」になっているはずです。

  9. hostnameコマンドで一時的に、ホスト名を「test.students.com」に変更して、hostnamectlコマンドでホスト名を確認してください。

    hostname test.students.com
    hostnamectl status
    ※Static hostnameの箇所が「students.com」、Transient hostnameの箇所が「test.students.com」になっているはずです。

  10. studentユーザに戻ってください。

    exit

5.netstatコマンドとssコマンド

  1. カレントディレクトリをホームディレクトリに変更してください。

    cd
    pwd

  2. rootユーザに変更してください。

    su -

  3. netstatコマンドを使って、全てアクティブなソケットで、ホスト・ポート・ユーザなどの名前を解決せずに、数字アドレスでTCPソケットに限定して確認してください。

    netstat -tan

  4. netstatコマンドを使って、ルーティングテーブルを確認してください。

    netstat -r　※routeコマンド同じ結果となります。

  5. ssコマンドを使って、全てアクティブなソケットで、ホスト・ポート・ユーザなどの名前を解決せずに、数字アドレスでTCPソケットに限定して確認してください。

    ss -alnt

  6. studentユーザに戻ってください。

    exit

6.pingコマンド

  1. カレントディレクトリをホームディレクトリに変更してください。

    cd
    pwd

  2. rootユーザに変更してください。

    su -

  3. pingコマンドを使って、www.google.co.jpに対して、オプション指定で10回、ICMPパケットを送信してください。

    ping -c 5 www.google.co.jp

  4. pingコマンドを使って、www.google.co.jpに対して、オプション指定で3秒間隔、ICMPパケットを送信してください。確認後、[Ctrl]+[C]で停止します。

    ping -i 3 www.google.co.jp
    [Ctrl]+[C]

  5. studentユーザに戻ってください。

    exit

  6. Windowsのコマンドプロンプト起動して、www.google.co.jpに対して、連続でICMPパケットを送信してください。確認後、[Ctrl]+[C]で停止します。

    ping -t www.google.co.jp
    [Ctrl]+[C]

7.tracerouteコマンドとtracertコマンド

  1. カレントディレクトリをホームディレクトリに変更してください。

    cd
    pwd

  2. www.google.co.jpに対してtracerouteコマンドを実行してください。

    traceroute www.google.co.jp

  3. Windowsのコマンドプロンプトを起動して、www.google.co.jpに対してtracertコマンドを実行してください。

    tracert www.google.co.jp
