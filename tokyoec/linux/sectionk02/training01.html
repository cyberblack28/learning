2.DNSサーバの構築

DNSサーバの構築テキスト

DNSサーバの構築テキスト

CentOS7_PROXY_DNSの仮想マシンで実行すること

  1. dnsmasqの稼動確認
    systemctl status dnsmasq.service
  2. dnsmasqの停止
    systemctl stop dnsmasq.service
    systemctl status dnsmasq.service
  3. dnsmasqの無効化
    systemctl mask dnsmasq.service
    systemctl status dnsmasq.service
  4. BINDをインストール
    yum install -y bind bind-utils
  5. BINDの自動起動設定
    systemctl enable named.service
    systemctl is-enabled named.service
  6. BINDの起動
    systemctl start named.service
    systemctl status named.service
  7. 「/etc/named.conf」を設定
    cp -p /etc/named.conf /etc/named.conf.org
    vim /etc/named.conf
    opsions設定の前に、以下の設定を行い、ネットワークグループの作成します。127.0.0.1と192.168.56.0/24を追加します。
    acl localnet {
  127.0.0.1;
  192.168.56.0/24;
};

    optionsのlisten-on portとallow-queryの設定を変更します。
    listen-on port 53 { 127.0.0.1; 192.168.56.28; };
allow-query { localnet;};

    最終行に以下、ゾーンファイルの設定（赤字）を追加します。
    zone "example.com" {
  type master;
  file "internal.zone";
};
zone "56.168.192.in-addr.arpa" {
  type master;
  file "internal.rev";
};
    Esc + :wq
  8. nmcliコマンドで「/etc/resolv.conf」に自身のIPアドレスを設定
    nmcli con mod enp0s8 ipv4.dns 192.168.56.28
  9. NetworkManagerを再起動して、設定を反映します。
    systemctl restart NetworkManager
    systemctl status NetworkManager
  10. 「/etc/resolv.conf」を確認
    cat /etc/resolv.conf
  11. 192.168.56.28以外のIPアドレスがある場合はコメントアウト
    vim /etc/resolv.conf
    # Generated by NetworkManager
#nameserver 192.168.1.1
#nameserver 192.168.0.1
nameserver 192.168.56.28
    Esc + :wq
  12. vim /var/named/internal.zone 
    $TTL 86400
@       IN  SOA  ns.example.com.  root.example.com.( 
          2016072601  ; serial              
          21600       ; refresh after 6 hours
          3600        ; retry after 1 hour
          604800      ; expire after 1 week
          86400 )     ; minimum TTL of 1 day
;
        IN  NS    ns
;
ns      IN  A     192.168.56.28
www     IN  A     192.168.56.28
ftp     IN  A     192.168.56.29
    Esc + :wq
  13. vim /var/named/internal.rev 
    $TTL 86400
@         IN  SOA  ns.example.com.  root.example.com. (
            2016072601  ; serial
            21600       ; refresh after 6 hours
            3600        ; retry after 1 hour
            604800      ; expire after 1 week
            86400 )     ; minimum TTL of 1 day
; 
    IN  NS    ns.example.com.
28	IN  PTR  www.example.com.
29	IN  PTR  ftp.example.com.
    Esc + :wq
  14. BINDの再起動
    systemctl restart named.service
    systemctl status named.service
  15. nslookupコマンドで確認（正引）
    nslookup
    ＞ server 192.168.56.28
    ＞ www.example.com
  16. nslookupコマンドで確認（逆引）
    nslookup
    ＞ server 192.168.56.28
    ＞ 192.168.56.28
    ＞ 192.168.56.29
    ＞ exit
  17. 外部からも名前解決できるようにfirewalldでdnsを解放
    firewall-cmd --add-service=dns --zone=public --permanent
  18. 設定を反映
    firewall-cmd --reload
